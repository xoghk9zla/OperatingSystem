***운영체제 20장 페이징 : 더 작은 테이블(ppt263~)***

~페이징의 문제점 2.
페이지 테이블의 크기가 너무 크다.(많은 메모리 공간을 차지한다.)~

단순하게 페이지 크기를 증가시키면, 페이지 테이블의 크기가 작아지지만,
내부단편화가 생긴다는 문제때문에, 메모리 낭비가 발생하므로 보편적으로 4 또는 8KB의 페이지를 사용한다.

프로세스당 하나의 테이블을 두되, 상위 2비트에 세그먼트 비트를 추가하여, 각 세그먼트의 페이지 테이블을 구별하는 방식이 메모리 낭비의 해결방법이 될 수 있지만, 이것도 문제가 좀 있다.

문제점
1.유연성이 없는 세그멘테이션을 사용해야 한다는 점.
2.힙의 경우에는 여전히 페이지 테이블의 메모리 낭비가 발생

그래서 등장한 기법이 멀티 레벨 페이지 테이블이다.
페이지 디렉터리라는 자료 구조를 바탕으로 할당여부와 위치를 파악한다.
PTE(페이지 테이블 데이터)의 상위레벨 개념으로 PDE라는 항목을 추가한다.
(PDE가 유효하다는 것은 PTE의 valid bit가 최소 하나는 1이라는 것)
그러므로 PDE가 유효하지 않을 경우 실제로 할당되지 않은 공간이라는 것을 알 수 있다.

단점은 실제로 선형적 페이지 테이블(PTE만 사용하는 방식)은 한번의 접근으로 주소 정보를 탑재할 수 있지만, 멀티 레벨 페이지 테이블은 PDE의 접근, PTE의 접근 두 번을 통해 정보를 탑재하므로, TLB미스 시 두 배가량의 시간이 걸린다.
