***운영체제 18장 페이징 : 개요(ppt 163)***
***ppt 172 중요한 내용이니 꼭 이해해야함***

물리 공간을 동일 크기의 조각으로 분할하여 단편화를 최대한 줄이는 방법.

페이징의 장점
-힙, 스택 영역이 어느방향으로 커지는가, 어떻게 사용되는가 가정을 하지 않아도 된다.
 (프로세스의 주소공간 사용방식에 상관 없이 효율적으로 주소공간 개념을 지원하기 때문, 
  유연성)
-빈 공간 관리가 단순해진다.(동일 크기로 나누어진 물리공간)



페이징을 하기위해서는 페이지 테이블이 필요하다.

가상 페이지 번호(VPN)을 물리 페이지 번호(PPN)으로 바꿔주어야 하기 때문.

ex)21이라는 가상 주소는 이진수 010101이다.

여기서 가상 페이지 번호가 각각 물리 페이지 번호로 매핑이 되어있으며,
(페이지 테이블이 이 정보를 가지고 있다.)
가상 페이지 번호가 0~3이라면, 2개의 비트로 모두 표현이 가능하기때문에,
상위 2개의 비트를 가상페이지 번호로 사용한다.

그러므로 010101 -> (01)(0101)로 나누어 지며 상위 2개 비트는
가상페이지 번호, 하위 4개 비트는 페이지 내 몇번째 바이트를 접근하는지 보여준다.
(이 예시는 1번 페이지에 5번째 바이트를 접근한다!)

이 때 01이라는 가상페이지 번호를 매핑 되어있는 물리페이지 번호로 변환하여
실제 물리 메모리에 접근한다.


** 페이지 테이블은 실제 어떤 비트를 가지고 있는지,

자세한 정보는 ppt 222~223 참조
valid bit -> 유효한 데이터인지
protection bit -> 접근 가능한 데이터인지
present bit -> 현재 페이지가 물리메모리에 있는지, 디스크에 있는지
dirty bit -> 페이지 정보가 변경 되었는지
reference bit ->페이지에 접근이 되었는지
